A05:2025 WEB · A05:2023 API · 38 CWEs
Injection
Concept : envoyer du code à la place de données. L'application ne distingue pas les données utilisateur des instructions — l'attaquant exploite cette confusion pour exécuter des commandes SQL, OS, LDAP…

Exemple classique : le champ "nom d'utilisateur" reçoit admin' OR '1'='1 — la base de données exécute la condition comme vraie et retourne tous les enregistrements.
⚠ Attaque
SQL injection
SELECT * FROM users
WHERE login = 'admin' OR '1'='1'
AND pass = '' OR '1'='1';
-- résultat : bypass auth complet
✓ Défense
Requêtes paramétrées (prepared statements)
ORM avec échappement automatique
Validation stricte du type des entrées
Principe du moindre privilège en DB
WAF en complément (pas en remplacement)
🔑
A01:2025 WEB · API1:2023 API · 40 CWEs · #1 le plus répandu
BOLA — Broken Object Level Auth
Concept : accéder aux données d'un autre utilisateur en changeant un identifiant dans la requête. L'API expose des endpoints avec des IDs — si elle ne vérifie pas que l'utilisateur connecté possède cet objet, n'importe quel utilisateur peut voir ou modifier les données des autres.

Aussi appelé IDOR (Insecure Direct Object Reference). C'est la vulnérabilité #1 des APIs en 2023.
⚠ Attaque
Accès non autorisé
GET /api/invoices/1042
// ma facture ✓

GET /api/invoices/1043
// facture d'un autre... 
// réponse 200 ← BOLA !
✓ Défense
Vérifier à chaque requête que l'objet appartient à l'utilisateur authentifié
Utiliser des UUIDs au lieu d'IDs séquentiels
Tests d'autorisation dans les pipelines CI/CD
Ne jamais se fier à l'ID dans le token seul
⚡
API4:2023 Unrestricted Resource Consumption · API6:2023 Sensitive Business Flows
Absence de Rate Limiting
Concept : une API sans limitation de débit peut être martelée automatiquement sans restriction. Résultats : brute-force de mots de passe, scraping massif, déni de service, exploitation de flux métier (achat automatisé de billets, spam de commentaires…).

Ce n'est pas un bug de code — c'est une absence de contrôle architecturale. L'OWASP 2023 API dédie deux catégories à ce problème.
⚠ Attaque
Brute-force login
POST /api/login × 10 000
{ "user": "alice",
  "pass": "password1" }
  "pass": "password2" }
  ...
→ 200 OK  ← mot de passe trouvé
✓ Défense
Rate limiting par IP et par compte (ex: 5 req/min)
Blocage progressif + CAPTCHA après échecs
Limites de pagination (max items par page)
Timeouts et quotas sur les ressources coûteuses
Monitoring des anomalies de volume
